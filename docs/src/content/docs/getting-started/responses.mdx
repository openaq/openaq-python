OpenAQ API and Python SDK responses share a consistent format. The SDK 
automatically transforms raw JSON from API into Python objects, enabling dot
notation  access. This design simplifies data retrieval and cuts down
boilerplate code. You can still access data as raw JSON or a standard dictionary
when needed.

## Response structure

All response objects use a standard top-level structure. The `results` attribute 
always contains a Python list (a JSON array). This rule applies even when 
fetching a single object via `get()`, or when filters yield no matches. If 
filters eliminate all matches, `results` returns an empty list.

```
├── headers
│   ├── x_ratelimit_limit: int | None
│   ├── x_ratelimit_remaining: int | None
│   ├── x_ratelimit_used: int | None
│   └── x_ratelimit_reset: int | None
├── meta
│   ├── name: str
│   ├── website: str
│   ├── page: int
│   ├── limit: int
│   └── found: int
└── results: list[]
```

Individual resource schemas reside in the Resources section of the
documentation.

## Accessing response data

The SDK supports three formats for response data.

### Response objects

By default, methods convert JSON responses into Python data objects. These 
objects support dot notation for quick property access. You can extract values 
directly using the preceding structure:

```py
# Access the page number
response.meta.page

# Access the remaining rate limits available for use
response.headers.x_ratelimit_remaining

# Access the first item in the results list
response.results[0]
```

### Dictionary

You can extract responses as a standard Python dictionary. Every response object
includes a `dict()` method. This method allows standard dictionary key lookup:

```py
response.dict()['results']
```

or the built-in `get()` method:

```py
response.dict().get('results')
```

Like the data object, the `results` key contains a Python `list`.


### JSON

To get the raw, unparsed response payload, use the `json()` method. This method
returns the raw JSON string:

```py
response.json()
```
