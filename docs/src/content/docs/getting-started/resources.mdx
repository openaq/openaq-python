The OpenAQ API follows a resource-based design where each endpoint returns a 
response corresponding to the endpoint. The OpenAQ Python SDK further simplifies
this by providing convenient resources from the client with methods to call the 
endpoints. All API responses through the SDK provide common features for reading
data as convenient Python objects, a dictionary, or a JSON string. 

```py
from openaq import OpenAQ

with OpenAQ() as client:
    response = client.instruments.get(instruments_id=1)
```

Here, `client.instruments` is the instruments resource, and .get() calls the
corresponding endpoint. Other resources follow the same or similar pattern
`client.locations.get()`, `client.sensors.list()`, and so on.
