As a derivative product of OpenAQ API, the OpenAQ Python SDK follows all [OpenAQ
API terms of use](https://docs.openaq.org/about/terms). As a first step, we
encourage familiarizing yourselves with these terms to be responsible data users
and respectful to publishers and the teams that have enabled and maintained open
data in the air quality space.

Next, the sidebar is an useful tool to navigate this documentation, and knowing
where what information lives will serve you better using the SDK:
1. **Getting started** - a good start if you're new to the SDK. Here, you'll learn
   the basics of the SDK to efficiently interact with the API. A tutorial to
   make your first query is also available in Quick start.
2. **Resources** - a decent skimming through the pages of this section will
   familiarize you with the way data is stored in OpenAQ database (i.e. via
   Resources) and how to use the SDK to get them. While inline documentation
   somewhat provides the same information, we recommend referring to these
   Resources pages as you write your code for clarity of methods and data
   definition.
3. **Advanced usage** - mostly customization options for more advanced users.
4. **Common workflows** - sample codes for common usage when utilizing the SDK to get air quality data with Python.
5. **Reference** - additional documentation that is less immediately relevant.
